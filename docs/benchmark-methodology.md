# Benchmark Methodology

## Principles

Benchmarks must be reproducible, scoped, and evidence-backed. A manifest is not a result, and a result is not a general performance claim.

## Required fields

A benchmark should identify workload, hardware profile, compiler configuration, metrics, software version, environment, and random seeds where applicable.

## Metrics

Current metrics include two-qubit gate count, two-qubit depth, pulse-duration estimate, and distinct SU(4) count.

## No unsupported claims

Do not merge benchmark comparisons unless the repository contains the manifest, execution method, result record, and evidence needed to reproduce them.
