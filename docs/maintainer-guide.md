# Maintainer Guide

## Triage process

Label issues by layer and request clarification when a claim lacks evidence.

## Release process

Before release, run tests, lint, type checks, package build dry-run, changelog updates, citation metadata updates, and documentation review.

## Reviewing pull requests

Check tests, docs, benchmark claims, dependency impact, third-party marks, and research-integrity language.

## Roadmap updates

Move items from planned to implemented only when code and tests are merged.

## Research-integrity review

Reject fabricated results, copied prose, false affiliations, and claims that turn scaffolds into completed algorithms.
